# 部署文档（中国内地访问 / 优先不备案）

## 产物形态

`web` 构建产物为纯静态文件：`web/dist/`。

构建命令：

```bash
cd web
npm run build
```

## 推荐路径 A：Cloudflare Pages（优先“快上线”）

适用：你希望尽快有公网 URL，且暂时不做 ICP 备案。

### 步骤（概要）

1. 将仓库推送到 Git 托管（GitHub/GitLab）。
2. 在 Cloudflare Pages 创建项目，选择该仓库。
3. 构建配置：
   - **Root directory**：`web`
   - **Build command**：`npm run build`
   - **Output directory**：`dist`
4. 绑定自定义域名（可选）。

### 中国内地访问说明（必须知情）

- 海外静态托管 **不一定**保证中国内地所有运营商/地区都稳定访问。
- **不做备案**意味着你通常无法使用内地 CDN 的“标准合规加速路径”，只能接受一定不确定性。

### 如果你发现内地访问不稳定

迁移到路径 B（国内静态托管 + 备案），或采用“双托管”（海外 + 国内镜像）——这需要额外运维成本。

## 路径 B：国内对象存储 / 静态网站托管 + ICP 备案（更稳）

适用：你的主要用户在中国内地，且希望长期稳定访问与更好的 CDN 体验。

特点：

- 周期更长：备案、实名、域名策略需要提前规划
- 但访问稳定性通常更好

> 具体云厂商控制台步骤会变化，本仓库不绑定单一厂商；落地时以你选择的云为准。

## 路径 C：Vercel / Netlify（备选）

与 Cloudflare Pages 类似：上线快，但内地可达性同样不保证。

## 安全与隐私

- 本应用默认 **不上传** 用户数据；导入导出由浏览器本地完成。
- 若你未来加后端：需要重新评估隐私条款与数据存储合规。

## 建议的发布检查清单

- `npm run build` 通过
- 手机端走一遍：导入 JSON → 改参数 → 看结果 → 导出 CSV
- 打开追溯：确认关键总量能展开
