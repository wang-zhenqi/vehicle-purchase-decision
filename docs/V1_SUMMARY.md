# v1 发布总结（购车决策量化计算器）

本文用于在 v1 发布后，复盘“做成了什么 / 怎么做的 / 有什么局限 / 下一步怎么演进”，并作为后续迭代的共同参照。

## 网页功能（你能在页面里做什么）

### 核心体验

- **移动端优先的四段导航**：车型 / 全局 / 假设 / 结果。
- **本地计算、无需登录**：所有数据默认保存在浏览器 `localStorage`，可以随时导入/导出。
- **可解释性（追溯树）**：任意方案的“总成本”可以展开为树状分解；点击追溯节点下方的路径标签，可跳转并高亮对应输入框。

### 输入页

- **车型**：支持添加多辆备选车（HEV / PHEV），录入能耗、价格构成、保值率；可选覆盖“年均保险/维保”。
- **全局**：录入共同影响所有车型的参数（年里程、油价/电价与充电占比、贷款参数、牌照费用等）。
- **假设**：录入“继续开宝来”的年度成本基线、大修风险准备金、机会成本收益率、简化的保险/维保规则、月供红线等。

### 结果页

- **宝来基线**：展示 5 年与 10 年基线总成本，并可一键打开追溯。
- **新车方案排行榜（5 年）**：
  - **卡片模式（默认）**：按“车型”对比，每车默认展示该车 **5 年总成本最低** 的组合；可在卡片内切换 **是否保留宝来** 与 **上牌策略**，并提供“恢复该车最优 / 全部恢复最优”按钮。
  - **表格模式**：平铺列出所有“车型 × 方案变体”，按 5 年总成本排序，便于密集浏览。
- **成本拆分展示**：
  - **购车成本（不含贷款利息）**
  - **贷款利息**
  - **合计（总成本）**
  - **残值（单独列出）**
- **敏感性分析**：对“选中的方案”做 ±X% 扰动，观察关键参数变化对 5 年总成本的影响区间。

### 工具栏（导入导出/示例/清空）

- **导出 JSON**：导出完整状态快照（含 `schemaVersion`、车型、全局、假设、导出时间）。
- **导入 JSON**：导入并合并默认值；对旧版导出中已删除字段会忽略；JSON 格式错误会提示。
- **导出 CSV**：导出简化结果表（更适合 Excel）。
- **加载示例**：加载 v1 默认示例数据（来自 `web/src/data/demo-state.json`）。
- **清空**：恢复默认状态（会清掉浏览器本地保存的当前数据）。

## 数据处理与计算逻辑（核心口径）

### 数据结构（高层）

- **状态**：`schemaVersion` + `cars[]` + `globals` + `assumptions`（均在前端存储与计算）。
- **输入路径规范**：表单字段统一带 `data-field="..."`，追溯节点的 `sources.path` 能精确定位输入项。

### 方案变体（上牌/持有组合）

v1 的方案变体由引擎按固定规则生成（`FULL_PLAN_GENERATION` 全量枚举），并结合家庭约束：

- **家庭策略约束**：**不放弃现有蓝牌指标**。
- **不包含**：蓝牌摇号、不包含深圳“蓝换绿”政策路径（作为家庭不会选择的策略）。

具体方案生成逻辑见 `web/src/engine/scenarios.ts`。

### 5 年总成本的口径

**口径摘要**：

- 5 年**总成本** = **购车成本（不含贷款利息）** + **贷款利息**
- **购车成本**中包含：落地价、牌照货币费用、机会成本、能耗、保险、维保
- 并扣减：期末残值、（若淘汰宝来）宝来卖出残值回收
- **不计入**：上牌等待/办理耗时/精力等“摩擦成本”的货币化

更完整的流程图与中间量见 `docs/CALC_PROCEDURE.md`。

### 追溯树（Trace）

- 计算引擎在输出每个关键总量时，同时输出一棵 `TraceNode` 树：
  - `label / unit / value / formula / children / sources`
- UI 通过 `TraceTree` 渲染：
  - 展开/折叠查看子项与公式
  - 点击来源路径跳转到对应表单字段并高亮

## 当前局限性（明确的“不做/做不到/会偏差”的地方）

### 模型假设与口径限制

- **大量参数依赖主观估计**：维保曲线、残值、风险准备金、机会成本等本质上是“决策假设”，并非客观真值。
- **贷款仅计利息**：本金偿还不计入“费用”（方便用“成本”角度比较，但与现金流口径不同）。
- **机会成本为线性近似**：当前用 `首付 × 年化收益率 × 年数`，未做复利/折现。
- **能耗简化**：
  - HEV：全里程按油耗计算
  - PHEV：按“用电里程占比”拆分油段/电段；未建模季节/路况/电池衰减
- **宝来残值模型简化**：用固定折旧倍率 `0.85^years`，可解释但不一定贴近真实市场。

### 场景覆盖限制

- **不做体验评分/舒适性货币化**：目前只做“成本”与少量风险/机会成本的代理输入。
- **不做自动抓取政策与车价**：所有政策费用、竞价价格仍靠手动输入。
- **不做延迟购买/等待情景**：已记录在 `docs/BACKLOG.md`，但 v1 主对比不混入该口径。

### 工程与产品形态限制

- 纯前端静态站：没有账号体系、没有服务端存储与协作分享链接。
- 数据在本机：更隐私，但换设备需要手动导出/导入。

## 可优化点（v1 后的优先级建议）

### 建议优先做（高性价比）

- **导出 CSV 的两种视图**：
  - “全量变体导出”（已支持）
  - “当前卡片选择导出”（更符合对比场景）
- **更强的导入校验**：对车型字段缺失/为 0 的情况，导入后集中提示并引导定位。
- **结果页对“卡片切换”的解释增强**：例如在卡片顶部提示“你当前正在看：保留宝来 / 竞价”。

### 可以后做（偏增强）

- 将“机会成本、残值”等拆成可选建模开关（仍保持追溯一致性）。
- 引入“按年曲线”的保险/维保模型，替代简单系数。
- 新增“延迟购买/政策不确定性”子面板（见 `docs/BACKLOG.md`）。

## 部署与运行说明（v1 已验证路径）

- **静态构建**：`web` 目录 `npm run build` 输出 `web/dist/`。
- **本地局域网手机访问**：
  - `npm run dev` 启动后使用终端输出的 **Network** 地址（同 WiFi）。
  - `vite.config.ts` 已设置 `server.host = true` 以便局域网访问。
- **静态托管**：
  - `docs/DEPLOY.md` 记录了 Cloudflare Pages / 国内静态托管等路径。
  - 本仓库根目录提供了 `package.json` 便于某些平台从仓库根检测并构建（最终产物仍是 `web/dist/`）。
