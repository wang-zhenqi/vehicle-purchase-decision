# PRD：购车决策量化计算器（Web）

## 背景与目标

用户希望把“是否换车 / 换什么车 / 何种牌照与持有策略”从直觉讨论，升级为**可复用、可解释、可调参**的决策工具：

- **不是**输出单一结论，而是输出“量表/计算器”：每个结果都能追溯到输入与公式假设。
- **以手机浏览器为主场景**：单手操作、信息密度可控、关键结果易读。
- **不需要登录与云端存储**：以本地存储 + 导入/导出为主。

## 用户与场景

- **主要用户**：家庭购车决策者（本人），需要同时考虑经济账、体验、政策/牌照路径不确定性。
- **关键场景**：
  - 录入多辆备选车（HEV/PHEV），对比不同持有/牌照组合的长期成本差异。
  - 快速回答：“在明确假设下，哪个方案更贵/便宜多少？贵在哪里？”

## 范围（第一期）

### In Scope

- 多车型管理（增删改）
- 全局参数（里程、油价/电价、贷款、牌照费用假设）
- 关键假设（宝来基线、风险准备金、机会成本、方案枚举开关）
- 结果：
  - 宝来基线（5年/10年）
  - 新车方案排行榜（默认5年总成本排序）
  - **追溯树（full trace）**：总成本展开为子项与中间量
- 导入导出：
  - JSON：完整状态备份/恢复
  - CSV：简化结果导出（便于 Excel）

### Out of Scope（第一期不做）

- 账号体系、云端同步、分享链接
- 自动抓取车价/参配/实时政策
- 完整的“体验评分模型”（可作为第二期：把体验货币化或加权评分）

## 功能需求

### 1) 输入层

- **车型卡片**：能源类型、能耗、价格构成、保值率；可选覆盖年均保险/维保。
- **全局参数**：油价、电价与充电占比、贷款参数、牌照费用参数。
- **关键假设**：宝来基线、风险准备金、机会成本、方案枚举开关。

### 2) 计算层

- 将“固定 6 方案”抽象为组合变量：
  - 是否保留宝来
  - HEV/PHEV 的牌照路径（迁入/竞价/摇号/绿牌/蓝换绿）
  - 保留宝来且发生指标迁移时，触发外地牌费用（中山）——以参数形式存在

### 3) 输出层

- **排行榜**：5年总成本、Δvs 宝来、月供、是否超过月供红线（用户可配）
- **追溯**：任意关键总量（例如5年总成本）可展开为树

### 4) 数据持久化

- 浏览器 `localStorage` 自动保存
- JSON 导入导出必须带 `schemaVersion` 以便未来迁移

## 非功能需求

### 移动端优先

- 主要导航在底部（大触控目标）
- 结果表格支持横向滚动；关键列固定可读性（后续可增强为“卡片模式”）
- 追溯使用全屏弹层（Dialog）

### 性能

- 纯前端计算，数据量小；仍需避免无意义的全量重渲染（后续可用 store 细分 selector）

### 可解释性

- 每个公式在追溯节点展示 `formula` 文本
- 对关键简化假设在 UI 文案中提示（例如：贷款本金偿还不计入费用）

## 验收标准（第一期）

- `web` 工程可 `npm run build` 通过
- 手机宽度（375px）下：导航可用、表单可读、结果表可横滑、追溯可读
- 任意新车方案可打开追溯，并能看到至少 2 层以上分解
- JSON 导出后可导入恢复同一状态

## 风险与免责声明

- 模型包含大量**假设**（维保曲线、残值、政策费用、机会成本等）。产品目标是辅助结构化思考，不是财务/税务/法律建议。
